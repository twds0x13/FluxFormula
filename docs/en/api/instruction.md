# Instruction

8-byte fixed-size instruction with explicit memory layout.

## Signature

```csharp
[StructLayout(LayoutKind.Explicit, Size = 8)]
public struct Instruction
```

## Fields

| Offset | Field | Type | Description |
|------|------|------|------|
| 0 | `OpCode` | `byte` | Operator's underlying byte |
| 1 | `Dest` | `byte` | Destination register (0-255) |
| 2 | `Arg0` | `byte` | Operand 0 |
| 3 | `Arg1` | `byte` | Operand 1 |
| 4 | `Arg2` | `byte` | Operand 2 |
| 5 | `Arg3` | `byte` | Operand 3 |
| 6 | `Arg4` | `byte` | Operand 4 |
| 7 | `Arg5` | `byte` | Operand 5 |
| 0 | `Raw` | `long` | Full 8-byte overlay view |

`Raw` shares offset 0 with `OpCode`. Reading `Raw` yields the entire instruction as a `long` value (for debug dumps); instruction execution uses the individual fields.

`ToString()` override: outputs `[{OpCode}] D={Dest} A=[{Arg0},{Arg1},{Arg2},{Arg3},{Arg4},{Arg5}]` format for debugging.

## Usage

Users generally do not operate on `Instruction` directly. It is generated by `FluxCompiler` and consumed by `FluxEvaluator` and `FluxExprCompiler`. Understanding the layout is useful for bytecode debugging.
