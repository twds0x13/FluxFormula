# IFluxBlobRegistry

Mod formula blob registry interface: automatically implemented by the source generator for each assembly containing formulas, enabling runtime scanning and loading. Core-layer interface with zero UnityEngine dependency.

## Role

In FluxFormula's **multi-mod architecture**, the base game is treated as the first mod. Each mod assembly's `BlobRegistry.g.cs` (SG-generated) implements this interface. `FluxBlobScanner` discovers all implementations via reflection at runtime and loads the corresponding blob data.

The `internal class BlobRegistry` in different assemblies does not conflict.

## Signature

```csharp
public interface IFluxBlobRegistry
```

## Members

| Member | Type | Description |
|--------|------|-------------|
| `EntryCount` | `int` | Number of formula entries in this registry. 0 means empty mod (no formulas) |
| `BlobKey` | `string` | Load key for the corresponding blob binary file. Used preferentially as an Addressables key; falls back to file path when Addressables is unavailable |
| `GetEntries()` | `BlobEntry[]` | Returns the compile-time-fixed offset table entries (DualHash64 → offset, length) |

## SG-Generated Example

The source generator produces the following (placed in the mod assembly):

```csharp
[assembly: FluxFormula.Core.FluxBlobRegistryAssembly]

internal sealed class BlobRegistry : IFluxBlobRegistry
{
    public int EntryCount => 42;
    public string BlobKey => "mod_formulas";

    public BlobEntry[] GetEntries() => new BlobEntry[]
    {
        new(new DualHash64(0x..., 0x...), offset: 0,   length: 156),
        new(new DualHash64(0x..., 0x...), offset: 156, length: 208),
        // ... EntryCount entries
    };
}
```

## FluxBlobRegistryAssemblyAttribute

`FluxBlobRegistryAssemblyAttribute` is an assembly-level marker attribute generated by the SG alongside `BlobRegistry`. `FluxBlobScanner` uses this attribute to quickly filter assemblies for scanning, avoiding a full `GetTypes()` traversal over all loaded assemblies.

```csharp
[AttributeUsage(AttributeTargets.Assembly, AllowMultiple = false)]
public sealed class FluxBlobRegistryAssemblyAttribute : Attribute
```

## FluxBlobScanner

`FluxBlobScanner` is a Unity-layer static scanner (in the `FluxFormula` assembly; no Addressables dependency). Its sole responsibility is discovery — loading is handled by the caller.

### DiscoverAll

```csharp
public static List<IFluxBlobRegistry> DiscoverAll()
```

Scans all loaded assemblies for types implementing `IFluxBlobRegistry`. Can be called multiple times (incremental scan — previously scanned assemblies are automatically skipped). Returns the list of newly discovered registries.

```csharp
// Typical usage: discover → load
var registries = FluxBlobScanner.DiscoverAll();
foreach (var r in registries)
{
    byte[] blobData = ResolveBlobData(r.BlobKey);
    FluxBlob.Load(blobData, r.GetEntries());
}
```

Skip logic:
- Registries with `EntryCount == 0` are automatically ignored
- `ReflectionTypeLoadException` is caught and logged as a warning; does not block the overall scan
- Assemblies must satisfy both conditions: `[FluxBlobRegistryAssembly]` marker and an `IFluxBlobRegistry` implementation

### ResetScanCache

```csharp
public static void ResetScanCache()
```

Resets the scan cache, forcing the next `DiscoverAll()` to re-scan all assemblies.

### LoadBlobFromFile

```csharp
public static byte[] LoadBlobFromFile(string key)
```

Loads blob data from a filesystem path. When not using Addressables, this can serve as the resolution mechanism for `BlobKey`.

- First tries `key` as a direct file path
- On failure, tries `Application.streamingAssetsPath/key`
- On success, automatically calls `BlobFormat.ExtractBlobData()` to extract the data segment
- Returns `null` if all attempts fail

### ScannedAssemblyCount

```csharp
public static int ScannedAssemblyCount { get; }
```

Number of assemblies scanned so far (diagnostic).

## Multi-Mod Data Flow

```
Compile time:
  FluxBlobBuilder.Build() → .blob binary file
  BlobRegistryGenerator (SG) → BlobRegistry.g.cs (compile-time offset table constants)

Runtime:
  FluxBlobScanner.DiscoverAll() → reflection discovers IFluxBlobRegistry impls
  → BlobFormat.ExtractBlobData() → pure data segment
  → FluxBlob.Load(blobData, registry.GetEntries()) → FluxBlobHandle
  → FormulaCache.Put() ×N

Unload:
  FluxBlob.Unload(handle) → FormulaCache.Remove() ×N → GCHandle.Free()
```
